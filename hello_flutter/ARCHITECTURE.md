# 系统架构设计文档

## 🎯 系统概述

本系统是一个基于 Flutter 的智能人体采集应用，通过整合相机、MediaPipe Pose 人体关键点检测和 IMU 传感器，实现自动化的标准人体照片拍摄。

## 📦 核心模块

### 1. 数据模型层 (models/)

#### DetectionMetrics
**职责**：封装所有检测指标和状态判断逻辑

**核心属性**：
- `isHumanComplete`: 人体完整性
- `distanceRatio`: 距离比例（0-1）
- `isPhoneLevel`: 手机是否水平
- `isStable`: 是否稳定
- `pitch/roll`: 俯仰角/翻滚角
- `landmarkCount`: 检测到的关键点数量

**核心方法**：
- `canAutoCapture`: 综合判断是否可以自动拍照
- `getStatusText()`: 获取当前状态提示文本
- `getStatusColor()`: 获取状态颜色（红/橙/绿）

**设计原则**：
- 不可变对象（const constructor）
- 单一职责：只负责数据封装和状态判断
- 纯函数：所有方法无副作用

### 2. 服务层 (services/)

#### CameraService
**职责**：管理相机生命周期和拍照功能

**核心功能**：
- 初始化后置摄像头（高分辨率）
- 拍照并保存到应用目录
- 资源释放

**技术要点**：
- 使用 `ResolutionPreset.high` 保证画质
- 异步初始化，避免阻塞 UI
- 时间戳命名文件，避免冲突

#### PoseDetectionService
**职责**：人体姿态检测和完整性分析

**核心功能**：
- 集成 Google ML Kit Pose Detection
- 实时检测32个人体关键点
- 分析人体完整性和距离比例

**工作流程**：
```
相机帧 → 图像转换 → ML Kit 处理 → 关键点分析 → 输出指标
```

**关键算法**：
1. **完整性检测**：检查7个关键部位（鼻子、双肩、双髋、双踝）
2. **距离估算**：计算人体高度占屏幕高度的比例
   - 理想范围：50% - 85%
   - < 50%：太远
   - > 85%：太近

**性能优化**：
- 流式模式（stream mode）实时检测
- 降低检测频率（10fps）平衡性能和实时性
- 当前使用模拟数据，待接入真实 ML Kit

#### IMUService
**职责**：读取 IMU 传感器数据，判断设备姿态和稳定性

**核心功能**：
- 读取加速度计数据（x, y, z）
- 计算俯仰角（Pitch）和翻滚角（Roll）
- 判断设备是否水平（±5°）
- 判断设备是否稳定（滑动窗口标准差）

**算法详解**：

**1. 角度计算**：
```dart
// 俯仰角（绕X轴旋转）
pitch = atan2(y, sqrt(x² + z²)) × 180/π

// 翻滚角（绕Z轴旋转）
roll = atan2(-x, z) × 180/π
```

**2. 稳定性检测**：
- 维护一个大小为10的滑动窗口
- 记录最近10次加速度大小
- 计算标准差：σ = √(Σ(xi - μ)² / n)
- 阈值：< 0.5 m/s² 视为稳定

**采样频率**：
- 加速度计：~50-100Hz（由系统决定）
- 回调频率：每次数据更新都触发

### 3. 界面层 (screens/ & widgets/)

#### CaptureScreen
**职责**：主控制器，协调所有服务和UI组件

**状态管理**：
```dart
enum CaptureStatus {
  idle,           // 空闲
  detecting,      // 检测中
  ready,          // 准备就绪
  capturing,      // 拍照中
  completed,      // 完成
}
```

**核心逻辑**：

**1. 初始化流程**：
```
启动 → 初始化相机 → 设置回调 → 启动IMU监听 → 进入检测状态
```

**2. 自动拍照流程**：
```
检测指标更新 
  ↓
检查 canAutoCapture
  ↓ true
状态变为 ready
  ↓
启动3秒倒计时
  ↓
每秒更新 countdownSeconds
  ↓
倒计时结束
  ↓
执行拍照
  ↓
状态变为 completed
```

**3. 状态流转图**：
```
idle → detecting ←→ ready → capturing → completed
                ↑                    ↓
                └──── restart ──────┘
```

**4. 双重保障**：
- 自动拍照：条件满足时自动触发
- 手动拍照：用户可随时点击按钮

#### StatusPanel
**职责**：实时显示四大检测指标

**UI结构**：
```
┌─────────────────────────────┐
│ 检测状态            [✓ 就绪] │
├─────────────────────────────┤
│ 👤 完整性    📏 距离        │
│   完整        65%           │
│                             │
│ 📱 姿态      ⚖️ 稳定        │
│   水平        稳定           │
├─────────────────────────────┤
│ 俯仰: 2.0° 翻滚: 1.5° 关键点: 32 │
└─────────────────────────────┘
```

**视觉反馈**：
- 绿色背景 + 绿色边框：指标合格
- 红色背景 + 红色边框：指标不合格
- 动态颜色根据 `metrics.getStatusColor()` 变化

#### GuidanceOverlay
**职责**：绘制引导遮罩和站位框

**CustomPaint 渲染流程**：

**1. 绘制顺序**：
```
半透明黑色遮罩（全屏）
  ↓
清除站位框区域（BlendMode.clear）
  ↓
绘制站位框（圆角矩形）
  ↓
绘制四个角标记
  ↓
绘制中心十字线
  ↓
绘制文字提示
  ↓
绘制倒计时（如果就绪）
```

**2. 站位框设计**：
- 位置：屏幕中央
- 尺寸：宽度80%，高度70%
- 样式：圆角矩形（radius=20）
- 颜色：动态（绿/橙/红）

**3. 角标记**：
- 长度：30px
- 粗细：6px
- 圆角端点（StrokeCap.round）

**4. 文字提示**：
- 顶部：状态提示（根据检测结果动态变化）
- 底部：固定提示"双脚分开与肩同宽，双手自然下垂"
- 阴影效果增强可读性

**5. 倒计时**：
- 位置：屏幕中央
- 背景：半透明黑色圆形
- 数字：大号字体（80px），动态颜色

**GPU加速**：
- 使用 CustomPaint 直接在 Canvas 上绘制
- shouldRepaint 精确控制重绘时机
- 避免不必要的 Widget 重建

### 状态驱动 UI

**响应式设计**：
- 所有 UI 组件都是 `metrics` 状态的函数
- `setState` 触发局部重建
- CustomPaint 通过 `shouldRepaint` 优化性能

**动画效果**：
- 倒计时数字变化（隐式动画）
- 颜色过渡（框架自动处理）
- 无额外动画控制器，保持简洁

## ⚡ 性能优化

### 1. 检测频率控制
- 姿态检测：10fps（每100ms）
- IMU 监听：原生频率（~100Hz）
- UI 刷新：按需（仅状态变化时）

### 2. GPU 加速
- CustomPaint 直接绘制到 Canvas
- 避免大量 Widget 嵌套
- 硬件加速的相机预览

### 3. 内存管理
- 及时释放相机资源（dispose）
- 关闭 ML Kit 检测器
- 取消传感器订阅

### 4. 异步处理
- 相机初始化异步
- 拍照操作异步
- ML Kit 处理异步

## 📝 总结

本系统采用分层架构设计，清晰分离关注点：
- **数据层**：纯数据模型，无业务逻辑
- **服务层**：封装原生 API，提供统一接口
- **UI 层**：响应式渲染，状态驱动

通过 MediaPipe Pose + IMU 的双重检测，确保拍摄质量。自动拍照逻辑综合考虑5大因素，模拟工业级产品体验。
